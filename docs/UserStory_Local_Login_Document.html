BrisConnect — User Story Documentation

Sprint 3 Local User Generated: April 27, 2026

1. User Story

As a Local User,

I want to log in to my account,

So that I can access local portal features.

2. Acceptance Criteria

  • Local login requires valid local credentials.
  • Invalid credentials are rejected with feedback.
  • Successful login redirects the user to the local portal.
  • Local-only routes are restricted from other user roles.
  • Session state is restored correctly after login.
  • The system shall provide secure, reliable, and efficient login for Local users.

3. User Steps (App Interaction)

  1. Open BrisConnect app → tap Local User Login on the home screen.
  2. Enter registered email and password in the login form.
  3. Tap the Login button to submit.
  4. If credentials are incorrect → error message displayed → user corrects and retries.
  5. If too many failed attempts → account locked → lockout message shown with retry guidance.
  6. If account is pending or disabled → access denied message shown.
  7. If credentials are valid → system verifies local user role and account approval.
  8. Redirected to the Local Portal dashboard with portal features unlocked.
  9. On next app open → restore key detected → session resumed automatically → user lands directly on Local Portal.
  10. Attempting to access local-only screens from another role → access blocked → redirected away.
